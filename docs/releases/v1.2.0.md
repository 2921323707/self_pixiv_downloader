# Pixiv Platform v1.2.0

## Release Type

Windows desktop delivery and maintenance release for the local-first Pixiv downloader workbench.

## Release Title

Pixiv Platform v1.2.0 - Windows Desktop Release

## Release Anchor

`v1.2.0` promotes the Windows Tauri app from local adaptation to a manually validated desktop release candidate. It keeps the existing downloader-first backend, Next.js workbench, and macOS source path, while making Windows NSIS the current default build target.

## Included Scope

- Windows NSIS installer build path as the default local Tauri build.
- Shared Web/Desktop default data location under `~/Downloads/Pixiv Platform/`.
- Windows-compatible download path validation and `USERPROFILE` home fallback.
- Windows native download-folder picker through the existing Settings storage control.
- Desktop logging on Windows at `%LOCALAPPDATA%\Pixiv Platform\desktop.log`.
- Pixiv Refresh fix in Settings: the frontend now resolves the Tauri invoke bridge more robustly and shows visible errors when the desktop bridge or command fails.
- Non-secret Rust desktop logs for Pixiv Refresh command entry, login-window creation, reuse, and creation failures.
- Existing macOS Tauri source branches remain in place, but the default checkout is Windows-oriented.

## Distribution Boundary

This release is the Windows desktop package. The Windows installer includes the Tauri shell and bundled frontend; users do not need Rust, Cargo, Node, npm, or TypeScript to run the installed app.

macOS `.app` / `.dmg` support remains source-compatible from the previous `v1.1.1` work, but rebuilding macOS packages from this checkout requires switching the Tauri build command and bundle targets back to macOS settings or using a future platform-specific config.

## Verification Baseline

Validated on Windows:

```text
cd src/frontend && npm.cmd run lint
cd tauri-app/src-tauri && cargo check
cd tauri-app && npm.cmd run build
```

Manual checks:

- Windows Web app starts and works.
- Windows Tauri app starts and works.
- Settings -> Pixiv connection -> Refresh opens the Pixiv login window and completes successfully after login.

Current Windows installer:

```text
tauri-app/src-tauri/target/release/bundle/nsis/Pixiv Platform_1.2.0_x64-setup.exe
```

## Release Notes

### Added

- Windows NSIS installer as the current default Tauri build artifact.
- Windows desktop log path documentation.
- Windows-specific download-folder picker path.

### Fixed

- Settings Pixiv Refresh on Windows Tauri release builds could fail to open the Pixiv login window because the frontend relied on a fragile Tauri internal invoke check.
- Windows absolute download paths are now accepted by settings validation.
- `~` expansion and default download root resolution now support `USERPROFILE` when `HOME` is unavailable.

### Changed

- Tauri package version updated to `1.2.0` so the generated installer filename matches the release tag.
- Desktop Pixiv Refresh failures now surface in the Settings UI and desktop log instead of failing silently.

### Security

- Pixiv `PHPSESSID`, DeepSeek API keys, and other secrets are not written to code, documentation, or logs.
- Pixiv Refresh logs only non-sensitive command/window status and cookie length metadata.

## Cleanup Notes

- Build artifacts under `tauri-app/src-tauri/target/`, Next output under `src/frontend/.next` / `src/frontend/out`, and dependency folders remain generated local artifacts and are not release source.
- Long-term dual-platform maintenance should split Windows and macOS Tauri build config to avoid changing `tauri.conf.json` back and forth.
