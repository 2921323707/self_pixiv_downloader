# Pixiv Platform v1.1.1

## Release Type

Mature first delivery release for the local-first Pixiv downloader workbench and macOS Tauri desktop app.

## Release Anchor

GitHub `v1.1.1` has been released. The user considers this version mature enough to serve as the first stable delivery baseline.

## Included Scope

- Downloader-first backend and frontend workbench from the v1.0.0 delivery baseline.
- macOS Tauri desktop app that reuses `src/frontend` and `src/backend`.
- Runtime local backend port injection instead of a fixed API port.
- Startup health check before creating the main desktop window.
- Desktop startup diagnostics and log file at `~/Library/Logs/Pixiv Platform/desktop.log`.
- Shared Web/App default download and SQLite location under `~/Downloads/Pixiv Platform/`.
- Old project `output/` recovery is no longer part of startup; use explicit backups if old local data must be preserved.
- Gallery delete flow and Gallery preview stability fixes.
- Built-in Tauri Pixiv login window for refreshing `PHPSESSID`.
- Explicit macOS ad-hoc signing for the app bundle with `bundle.macOS.signingIdentity = "-"`.

## Distribution Boundary

The current macOS package is ad-hoc signed, not Apple Developer ID signed, and not notarized.

Small-scale testers do not need Rust, Cargo, Node, npm, or TypeScript to run the packaged app. macOS Gatekeeper may still require the user to manually allow the app because the package is not notarized.

## Verification Baseline

Use these checks for the packaged app:

```text
cd tauri-app && npm run build
codesign --verify --deep --strict --verbose=2 "tauri-app/src-tauri/target/release/bundle/macos/Pixiv Platform.app"
hdiutil verify "tauri-app/src-tauri/target/release/bundle/dmg/Pixiv Platform_1.1.0_aarch64.dmg"
hdiutil attach -nobrowse -readonly "tauri-app/src-tauri/target/release/bundle/dmg/Pixiv Platform_1.1.0_aarch64.dmg"
ls -la "/Volumes/Pixiv Platform"
codesign --verify --deep --strict --verbose=2 "/Volumes/Pixiv Platform/Pixiv Platform.app"
hdiutil detach "/Volumes/Pixiv Platform"
```

## Debug Notes To Preserve

- If the app fails on startup, check `~/Library/Logs/Pixiv Platform/desktop.log`.
- If the main window does not appear, check the backend health path `GET /api/health`.
- If old images or settings are missing, inspect the shared SQLite path and any manual backups of old local data.
- If macOS reports that the app is damaged, run `codesign --verify --deep --strict` and confirm the app bundle contains `_CodeSignature/CodeResources`.
- Never log or document the full Pixiv `PHPSESSID`.
